Redis 缓存问题与分布式锁

Redis 缓存问题与分布式锁一、缓存问题1. 什么是缓存穿透？缓存穿透是由于 key 不存在导致的。 解决办法： 如果 key 在 redis 中没有数据，数据库也没有数据，我们就将这个不存在的 key 创建出来，并设置它的值为空。考虑到一种情况，后面 key 所对应的数据在数据库中有了数据，而此时 redis 中没有数据，数据不一致，为了避免这种情况，我们在将 redis 的 key 设置为空的同时，还会给它设置较短的过期时间。 2. 什么是缓存击穿？缓存击穿是由于 key 过期导致的，即 key 在 redis 中有数据，但是这个 key 过期了，所有请求都跑去获取数据库中的数据。 解决办法： 如果有 1000 个请求，原本查询 redis 而现在都去查询数据库，这样数据库压力就会很大。我们会只放一个请求去查询数据库，让剩下 999 个请求先等待，等那一个请求查询完，并将数据存入 redis 后，剩下的 999 个请求都去请求 redis。 3. 什么是缓存雪崩？缓存雪崩则是由于 大量 key 过期导致的。 解决办法： 让 redis 的 key 过期时间不一致，不要设置同一个时
