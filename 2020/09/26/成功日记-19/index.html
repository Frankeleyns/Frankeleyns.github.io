《经济学原理》

进度：第十二章 税制的设计

公司需要交两次税

首先，政府会根据公司的利润让公司交公司所得税，其次，当公司用利润向股东支付股息时，股东们要交个人所得税。

税收的平等

能力纳税原则：根据一个人所能承受的负担对这个人征税。由这个原则分出了横向平等和纵向平等。

横向平等：有相似支付能力的纳税人应该缴纳相等的量。

纵向平等：能力大的纳税人应该缴纳更大的量。

税制类型

比例的：所有纳税人都支付同样的收入比例。

累退的：高收入纳税人支付的税收量大，但只占他们收入的小部分。

累进的：高收入纳税人支付他们收入中较大的比例。

《穷查理宝典》

进度：查理十一讲 第六讲 一流基金的投资实践

基金投资

书中原文：

假设基金是赌徒，它们每年交给赌场荷官的费用是起始资金的3%，所有股票投资者将必须忍受这样的情况：他们每年赚的钱有一部分落进了荷官的口袋。这是不可避免的命运。同样不可避免的还有，在支付了荷官费用之后，正好有一半的投资者的收益率将会低于平均线，而这条平均线(未来)完全有可能落在一般和糟糕之间。

基金投资，无论你赚还是亏都必须缴纳一定费用（交易税，印花税之类的），这些固定费用养活了一批人，而投资者的收益却会降低，要想赚钱必须做到减少频繁交易，长期持有基金。

这篇演讲最核心的一句话就是——建议基金采用一种更有成效的办法，就是对少数几家广受赞誉而且名副其实的国内公司进行长期的集中投资。

对我们投资基金也可采取类似的方法：

  1. 投资收益排名较为靠前的基金，且经营时间最好超过五年。
  2. 长期持有（1年以上），连续定投。

《计算机网络原理》

进度：第五章：数据链路层与局域网

数据链路层服务

链路：通信链路连接的相邻节点的通信信道。

链路层数据单元：帧。

数据链路层主要功能：

  1. 组帧
  2. 链路接入：点对点链路、广播链路
  3. 可靠交付（可靠传输方法多用于高出错链路）
  4. 差错控制

差错控制

差错控制的四种基本方式：

  1. 检错重发
  2. 前向纠错
  3. 反馈校验
  4. 检错丢弃

典型的差错编码

1.汉明码

汉明距离：两个等长码字之间，对应位不同位数。

编码集的汉明距离：该编码集中任意两个码字之间汉明距离的最小值。

例如：给定一个编码集{000000，010101，101010，111111}

我需要传输编码101010结果途中产生了差错，变成了101110，然后拿101110和编码集中的所有编码进行比较，发现101110和101010只差1位不一样，它两汉明距离最小，于是我们将101110纠正为101010。

2.奇偶校验码：

奇校验：编码后码字中的”1“的个数为奇数

偶校验：编码后码字中的”1“的个数为偶数

3.循环冗余码CRC：

检错能力强，编码效率高，实现简单。

举例：CRC编码的生成多项式位G(x) = x^4 + x + 1，为10111001进行CRC编码。

  1. 多项式转化为二进制：

    x^4 + x +1 = 1 x^4 + 0 x^3 + 0 *x^2 + 1 *x + 1 *x^0，由此得出：10011**

2.在编码串10111001后面加0000，因为最高系数为4次方，得到：101110010000

3.用10111001000除以10011，余数为1001，CRC编码后的结果为：101110011001

二进制的除法过程采用的是 异或（相同的值为0，不同的值为1），比如：余10000和10011得出的结果为：00011

Java小技巧

使可变性最小化

Java类库包含许多不可变类，其中有String、基本类型的包装类、BigInteger和BigDecimal。这些不可变类比可变类更加易于设计、实现和使用。

使类要不可变，有五条原则：

  1. 不要提供任何会修改对象状态的方法(也称设值的方法，比如Setter)。

  2. 保证类不会被扩展。

  3. 声明所有的域都是final的。

  4. 声明所有的域都是私有的。

  5. 确保对于任何可变组件的互斥访问。

下面举一个例子：

public final class Complex {
    private final double re;
    private final double im;

    public Complex(double re, double im) {
        this.re = re;
        this.im = im;
    }

    public double realPart(){ return re; }
    public double imaginaryPart(){ return im; }

    public Complex plus(Complex c){
        return new Complex(re + c.re, im + c.im);
    }

    public Complex minus(Complex c){
        return new Complex(re - c.re, im - c.im);
    }

    public Complex times(Complex c){
        return new Complex(re * c.re - im * c.im,
                           re * c.im + im * c.re);
    }

    public Complex dividedBy(Complex c){
        double tmp = c.re * c.re + c.im * c.im;
        return new Complex((re * c.re + im * c.im) / tmp,
                               (im * c.re - re * c.im) / tmp);
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Complex)) return false;
        Complex c = (Complex) o;
        return Double.compare(c.re, re) == 0 && Double.compare(c.im, im) == 0;
    }

    @Override
    public int hashCode() {
        return 31 * Double.hashCode(re) + Double.hashCode(im);
    }

    @Override
    public String toString() {
        return "Complex{" +
                "re=" + re +
                ", im=" + im +
                '}';
    }

}

这个类表示一个复数（complex number，有实部和虚部）。里面有四个基本算术运算：加、减、乘、除。需要注意的是这些算术运算都是返回新的实例，而不是修改现有的实例。这种方式被称为函数的（functional）的方法，这些方法返回的是一个函数的结果，这些函数对操作数运算但不修改它。
